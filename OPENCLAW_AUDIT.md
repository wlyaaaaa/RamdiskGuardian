## 🔍 Detailed Audit Findings

### 1. Trigger: `LogonTrigger` → `BootTrigger`

**Before (Defective)**:
```xml
<Triggers>
  <LogonTrigger>
    <UserId>WLY\10979</UserId>
  </LogonTrigger>
</Triggers>
```

**Problem**: The gateway only starts when user `WLY\10979` logs in interactively. If the machine reboots unattended (e.g., after a Windows Update), the gateway will NOT start until someone physically logs in.

**After (Fixed)**:
```xml
<Triggers>
  <BootTrigger>
    <Delay>PT30S</Delay>
  </BootTrigger>
</Triggers>
```

**Fix**: Changed to `BootTrigger` with a 30-second delay to allow networking/Tailscale to initialize.

---

### 2. LogonType: `InteractiveToken` → `S4U`

**Before (Defective)**:
```xml
<Principal id="Author">
  <UserId>S-1-5-21-1536040469-1817842872-232824913-1001</UserId>
  <LogonType>InteractiveToken</LogonType>
</Principal>
```

**Problem**: `InteractiveToken` means the task runs in the interactive desktop session and **will flash a CMD/PowerShell window** every time. It also requires the user to be actively logged in.

**After (Fixed)**:
```xml
<Principal id="Author">
  <UserId>WLY\10979</UserId>
  <LogonType>S4U</LogonType>
  <RunLevel>HighestAvailable</RunLevel>
</Principal>
```

**Fix**: Changed to `S4U` (Service-for-User). This logon type:
- Runs the task under the user's identity **without requiring an active logon session**
- Does NOT prompt for a password (uses the stored Kerberos S4U token)
- Does NOT create any visible window on the desktop
- Is the recommended approach for headless background services on Windows

---

### 3. RunLevel: Not Set → `Highest`

**Before**: No `<RunLevel>` element present (defaults to `Limited`).

**Problem**: OpenClaw Gateway needs elevated privileges for:
- Binding to port 18789
- Tailscale funnel operations
- Network interface management

**After (Fixed)**: `RunLevel` set to `Highest` (runs with full administrator privileges).

---

### 4. No Restart-On-Failure Policy (New)

**Before**: No restart policy configured. If the gateway crashes, it stays down until next reboot.

**After (Fixed)**:
- Restart interval: 60 seconds
- Maximum restart attempts: 3
- Combined with `StartWhenAvailable`, ensures missed starts are caught up

### 5. VBScript Hidden Wrapper (New)

Even with `S4U` and `Hidden` task settings, some Windows versions can briefly flash a console window. The fix adds a **VBScript wrapper** (`openclaw_run_hidden.vbs`) that uses `WScript.Shell.Run` with `windowStyle=0` as an additional insurance layer:

```vbscript
shell.Run """C:\Users\10979\.openclaw\gateway.cmd""", 0, False
```

This guarantees **absolutely zero window visibility** across all Windows versions.
